API Push

Funcionalidade
Possibilita que aplicações web recebam mensagens enviadas a elas de um servidor, indepententemente de estarem ou não em primeiro plano, ou até mesmo carregada, em um agente de usuário. Exemplo de Notificação Push

Características importantes
- Necessita de um service worker ativo, para receber mensagens push.
- Cada assinatura é única para um service worker.
- Endpoint para a assinatura => URL única, mantida em segredo

Uso de recursos
- Ativação de um service worker para entregar mensagens push:
pode resultar em um aumento de uso de recursos, particularmente de bateria.

Uso de recursos
=> Firefox: permite um número limitado (cota) de mensagens push para serem enviadas para uma aplicação *.
=> Chrome: não aplica nenhum limite, mas requer que cada mensagem push exiba uma notificação.

Compatibilidade de navegadores
- Edge v17+
- Firefox v44+
- Chrome v44+
- Opera v42+
